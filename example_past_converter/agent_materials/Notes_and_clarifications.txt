Canonical SurLab: axis 0 = time, axis 1 = ROIs/features (Python (n_time, n_unit); Matlab [time × …]).
Suite2p (n_ROI, n_time) must be transposed on ingest — stated in doc clarifications table and stage 3/4 notes on data rows.

B — X vs Y schema
Fixed the contradiction: X_* = axis 0, Y_* = axis 1. For traces/behTSeries: X_dim=time, Y_dim=ROI/features, X_size = n_time, Y_size = n_ROI/features (matches metadata rows). Table rows and doc schema section updated.

C — Resampling
Native frame times in _timestamps are canonical. No required resampling; optional sample_frequency__Hz is descriptive only. Reference-script 20 Hz upsampling is pipeline-specific, not SurLab.

D — zero_time (Tricolor)
Canonical for Tricolor/grating: zero_time = primary calcium <datatypeID> (e.g. GCaMP7f_traces) with t = 0 at imaging acquisition start; keep native timestamps (negative pre-stim allowed).
Do not silently re-zero to first stimulus unless the dataset README documents that and timestamps/trials are handled consistently. The reference script’s “first stim = 0” is not the default SurLab policy.

E — Session folder naming
Folder = {animal_ID}_{session_ID} with one underscore, literal join. session_ID may contain dots (e.g. exp101.a1.20260204.gratings). Overlap with animal_ID text is fine — they stay separate columns.

F — Multiple trace streams
Separate stream per indicator/channel product, each with its own <datatypeID> and sessionInfo flag: e.g. GCaMP7f_traces, tdTomato_traces, mCherry_traces.
Not one combined array for dual-channel + tricolor. F / F_chan2 → separate streams when processed separately; tricolor classes belong in ROI metadata, not as extra streams unless each is its own recording.

G — Required sessionInfo fields
Still required in the exported sessionInfo CSV. Ingest may fill from dataset_config / sidecar before write. New table row (session_metadata_provenance) documents this (not a real column).

G2 — trialInfo.correct
Changed to recommended in the table; doc says omit for passive/grating demos without operant outcomes.

H — Conversion table location
Authoritative: repo root conversion_table.csv. for_cursor/conversion_table.csv is a handoff copy (synced now). Doc updated.

I — .mat vs .npz
Both valid; prefer .npz for Python export (doc + table notes). any further questions before you code?
